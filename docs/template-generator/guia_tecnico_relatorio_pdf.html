Gerador de Relatório PDF — LeãoCorp Analytics

O relatório é gerado em Python puro — não é HTML/CSS/JS. Dois motores trabalham juntos: matplotlib desenha os visuais e reportlab monta e exporta o PDF.

Python 3.10+ reportlab matplotlib numpy
Fluxo de geração
JSON da análise → Python extrai dados → matplotlib gera PNGs → reportlab monta layout → PDF A4

O truque central: fig_to_img() converte cada figura matplotlib em PNG dentro de um io.BytesIO (arquivo virtual em RAM, sem tocar o disco), e reportlab embute esse buffer via Image(buf, width=..., height=...).

Gráficos vetoriais
Funil trapezóide, donut, barras empilhadas e jornadas com caixas+setas. Suporta fundo transparente (alpha=0) para integrar com o layout sem "caixas brancas".

Layout e exportação PDF
Sistema de story: você empilha elementos (Paragraph, Table, Image, Spacer) numa lista e doc.build() distribui automaticamente em páginas A4 respeitando margens.
Funções de gráfico
make_funnel_chart(w_in, h_in)
Desenha trapézios empilhados proporcionais à contagem de leads. Cada fase tem largura = min_w + (max_w − min_w) × frac, onde frac = leads_fase / leads_topo. O fundo da fase N é a largura da fase N+1 — isso cria o efeito trapezoidal automaticamente.
make_journey_chart(w_in, h_in)
Substitui a tabela de sequências ilegível. Usa FancyBboxPatch para caixas arredondadas e ax.annotate() para setas. Taxas posicionadas em by + box_h + 0.025 (acima da caixa) para não colidir com texto interno.
make_daily_chart(w_in, h_in)
Barras empilhadas com bottom=daily_ia para sobrepor a série Híbrida sobre a série IA pura. Dados do campo novo daily_track_volume.
make_donut_chart(sz)
Gráfico de pizza com wedgeprops={'width': 0.52} que cria o anel. Texto central inserido via ax.text(0, 0, ...).
Helpers de layout reportlab
sec_label(text, w)
Título de seção com barra verde via ('LINEBEFORE',(0,0),(0,0), 2.5, C['green']) no TableStyle.
kpi_cell(label, val_p, sub)
Card KPI em 3 linhas: label 6.5pt / valor 20pt bold / subtexto 6.5pt. Fundo C['light'] via BACKGROUND no estilo.
final_panel(title, total, ia_pct, ...)
Painel dos campos novos. A "barra de progresso" é uma Table de 1 célula com largura = int(pct * 0.75) pt e BACKGROUND colorido — não existe barra nativa no reportlab.
insight_box(bg_hex, acc_hex, ...)
Caixa com fundo tênue + borda lateral colorida (LINEBEFORE). O padding assimétrico cria o visual de "card com accent".
Campos novos do JSON → elementos do PDF
daily_track_volume NOVO → Gráfico de barras empilhadas por dia (make_daily_chart). Mostra pico do dia 17/04 com 19 atendimentos.
final_stages_detected NOVO → Identificação automática dos estágios finais (HANDOFF_AG, HANDOFF_EB) com converted_count. Alimenta os painéis de "Conversão para etapas finais".
final_stage_conversion_by_track NOVO → Breakdown IA pura vs Híbrida por etapa final. Alimenta as barras de progresso dentro de final_panel(). Revelou que híbrida converte 5.9× mais em e-book.
Correções de design aplicadas
✕
Antes: barras horizontais independentes
12 etapas com percentuais soltos (16.7% a 81.1%) sem hierarquia visual. O cérebro não percebia a "jornada de perda".
✓
Depois: funil trapezoidal com macro-fases
5 fases agrupadas por significado (Contato → Qualificação → Oferta → Coleta → Conversão). Trapézios com largura proporcional aos leads: 54 → 33 → 31 → 20 → 16. A perda é imediatamente visível.
✕
Antes: tabela de strings "PINV>SAUD>QUEIXA>APRO>..."
Parecia log de programador. Gestor de clínica não lê logs; precisa ver jornadas em português.
✓
Depois: caixas + setas com taxas de conversão
3 caminhos nomeados em português ("Caminho principal — agendamento") com caixas arredondadas, setas e taxas acima de cada transição. FancyBboxPatch + ax.annotate() do matplotlib.
✕
Antes: colunas sem largura fixa → "RECORRENTE TE" e "CONVITE\nAVAL."
reportlab distribuía colunas automaticamente, quebrando textos de forma imprevisível.
✓
Depois: colWidths explícitos em todos os Table
CW = [60, 12, 60, 40] — a soma (172pt) é menor que half2 (~261pt), deixando folga. Labels abreviados garantem que nenhum texto ultrapasse a célula.
